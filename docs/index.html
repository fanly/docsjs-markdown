Why docsjs-markdown?

Dual Entry Architecture

Main entry for HTML conversion, sub-entry /docx for direct DOCX to markdown conversion.

Rule-Based Conversion

Customize conversion per tag and keep your markdown style guide stable across products.

CLI & Pipeline Friendly

Use in CI and batch jobs for consistent markdown generation with optional frontmatter and report metadata.

Recommended Pair

Combine with @coding01/docsjs to build a full Word/DOCX to markdown publishing chain.
